# PWA + Offline-First + Push Notifications Design

**Date**: 2026-03-02
**Status**: Approved

## Goal

Make EventTara a Progressive Web App with a progressive offline-first strategy and Web Push notifications. Phase 1 focuses on installability, offline shell with cached data, and push notifications for key events.

## Approach

**Serwist + web-push + Supabase Edge Function** (Approach A). Serwist handles the service worker and caching layer. VAPID-based Web Push for notifications. Supabase DB webhook + Edge Function triggers push delivery on notification INSERT.

No Turbopack trade-off — the project already uses Webpack for dev.

## 1. Service Worker & Offline Shell (Serwist)

### Setup

- Install `@serwist/next` (runtime) + `serwist` (dev)
- Wrap `next.config.mjs` with `withSerwist()` — existing config preserved
- Service worker source: `src/app/sw.ts`
- Compiled output: `public/sw.js` (gitignored)
- tsconfig: add `@serwist/next/typings` to `types`, `webworker` to `lib`, exclude `public/sw.js`

### Caching Strategy

| Content                           | Strategy                      | Notes                              |
| --------------------------------- | ----------------------------- | ---------------------------------- |
| App shell (layouts, CSS, JS)      | Precache                      | Auto-generated by Serwist          |
| `/~offline` page                  | Precache                      | Fallback for failed navigations    |
| `/api/events` (list)              | Stale-while-revalidate        | First page cached                  |
| `/api/events/[id]` (detail)       | Network-first, cache fallback | Cached on first visit              |
| `/api/bookings`, `/my-events`     | Network-first, cache fallback | User-specific data                 |
| `/api/notifications/unread-count` | Stale-while-revalidate        | Badge count                        |
| Static assets (images, fonts)     | Cache-first                   | Already have 1yr immutable headers |
| Auth routes, mutations, `/admin`  | Network-only                  | Never cached                       |

### Offline Fallback

When a navigation request fails (no network) and no cached page exists, serve `/~offline` — a branded page indicating offline status with a note that cached content may still be available.

## 2. Push Notifications

### New Table: `push_subscriptions`

```
push_subscriptions
  id: uuid (PK, default gen_random_uuid())
  user_id: uuid (FK -> users.id, ON DELETE CASCADE)
  endpoint: text (unique)
  keys_p256dh: text
  keys_auth: text
  created_at: timestamptz (default now())
```

User creates the migration. RLS: users can INSERT/DELETE their own subscriptions.

### VAPID Keys

- Generated once via `npx web-push generate-vapid-keys`
- `NEXT_PUBLIC_VAPID_PUBLIC_KEY` — public, exposed to client
- `VAPID_PRIVATE_KEY` — secret, server-side only
- Also stored as Supabase Edge Function secrets

### Push Triggers

Only these notification types trigger a browser push:

- `booking_confirmed`
- `event_reminder`
- `badge_earned`
- `border_earned`

Feed activity (`feed_like`, `feed_repost`) stays in-app only.

### Client-Side Flow

1. After login, check `'PushManager' in window`
2. Show custom opt-in UI (not raw browser permission dialog)
3. On opt-in: `registration.pushManager.subscribe()` with VAPID public key
4. POST subscription to `/api/push-subscriptions`

### Server-Side Flow

1. `notifications` table INSERT triggers Supabase DB webhook
2. Webhook calls Supabase Edge Function (`send-push`)
3. Edge Function checks notification type (skip if not in push triggers list)
4. Fetches user's push subscriptions from `push_subscriptions`
5. Sends Web Push via VAPID protocol
6. Deletes expired subscriptions on 410 Gone response

### Service Worker Push Handler

- `push` event: parse payload, show native notification with title, body, icon, href
- `notificationclick` event: open/focus the app at the notification's href

## 3. Install Prompt

### Manifest Updates (`public/site.webmanifest`)

- Add `categories: ["sports", "lifestyle", "travel"]`
- Add maskable icon variant (192x192)
- Verify `start_url`, `scope`, `display: standalone` are set

### InstallPrompt Component

- Listens for `beforeinstallprompt` (Chrome/Android)
- Branded dismissible banner at bottom of page (above mobile nav)
- iOS detection: shows "Tap Share -> Add to Home Screen" instructions
- Dismiss state in `localStorage`, don't nag repeatedly
- Only shown after 2+ visits (visit counter in `localStorage`)

## 4. Offline Indicator

- Small banner at top of app when `navigator.onLine === false`
- Text: "You're offline — showing cached data"
- Disappears when connectivity returns
- Listens to `online`/`offline` window events

## 6. User Responsibilities

- Create Supabase migration for `push_subscriptions` table + RLS policies
- Set up Supabase DB webhook (notifications INSERT -> Edge Function)
- Generate VAPID keys and add to `.env.local`
- Store VAPID keys as Supabase Edge Function secrets
- Deploy Edge Function (`supabase functions deploy send-push`)

## 7. Out of Scope (Phase 2+)

- IndexedDB for persistent offline data
- Background sync for queued mutations (offline booking)
- Periodic background sync for data freshness
- Offline check-in with sync
